# Phase working notes

One file per phase is added here **as we build it** (e.g. `phase-1-consumption-capture.md`),
capturing the actual implementation: entities created, helper definitions, decisions made
mid-build, what was tested, and anything that surprised us.

The roadmap lives in [../project-plan.md](../project-plan.md); these are the build logs that
back it. Convention per file:

- **What we built** — concrete entities/helpers/automations (with IDs).
- **How to reproduce** — YAML package or MCP/UI steps.
- **Decisions & surprises** — why we deviated, gotchas hit.
- **Verification** — how we confirmed it works.
